CloudAnalyzer / Public Demo

Stanford Bunny, browser-side inspection, no custom backend.

This Pages build publishes a static ca web bundle generated from the public Stanford Bunny mesh. The demo keeps progressive loading, point picking, heatmap thresholding, and trajectory inspection active while serving only HTML and JSON.

The source point cloud is derived from the public bunny mesh. The reference cloud and trajectories are generated during the Pages build to demonstrate CloudAnalyzer inspection flows on a small, reproducible asset.

What This Shows

  • Progressive chunk loading on a static host.
  • Heatmap thresholding without a Python server.
  • Point picking with XYZ and nearest-reference distance.
  • Trajectory overlay and linked error timeline inspection.

Build It Locally

After installing the package, rebuild the public demo bundle with:

python scripts/build_public_demo.py --output docs/demo/stanford-bunny

Data Note

The Pages bundle ships with a data attribution file generated next to the demo. The Stanford repository allows research use and free redistribution with credit, and keeps its own non-commercial restriction on the source assets.
